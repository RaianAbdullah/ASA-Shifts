# System Architecture

**Status: Stage 1 — Foundation**

---

## Overview

ASA Workforce uses a mobile-first client-server architecture. The React Native mobile app communicates exclusively through the secured REST API. There is no direct database access from the mobile application.

## Security Boundaries

1. **Mobile ↔ API**: HTTPS only. JWT Bearer token in Authorization header.
2. **API ↔ PostgreSQL**: Private network. Application-user credentials only.
3. **API ↔ Redis**: Private network. Password-protected.
4. **No direct DB access from mobile.**

## Key Architectural Decisions

- **Stateless API**: No HttpSession. All state in short-lived JWT + rotating refresh token.
- **Server-side time**: Attendance check-in time generated by the backend — never trusted from mobile.
- **Deny-by-default**: All endpoints require authentication unless explicitly permitted.
- **Public UUID**: API exposes UUID (publicId) not internal integer IDs.
- **Soft deletes**: Historical attendance and audit records are never permanently deleted.
- **Context path /api**: All endpoints under `/api` prefix for proxy routing.
